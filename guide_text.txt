SECTION 04
The Viral "10× Engineer" Workflow
Orchestration Template
Feb 25 2026 — Currently the most copied and highest-performing version in the community.
# IIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIII
# Workflow Orchestration
# IIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIIII
## 1. Plan Mode Default
- Enter plan mode for ANY non-trivial task (3+ steps or architectural decisions)
- If something goes sideways, STOP and re-plan immediately
- Write detailed specs upfront to reduce ambiguity
## 2. Subagent Strategy
- Use subagents liberally to keep main context clean
- One focused task per subagent
- Offload research, exploration, and parallel work
## 3. Self-Improvement Loop ← Boris-core
- After ANY correction: update tasks/lessons.md with the pattern
- Write rules that prevent the same mistake forever
- Review relevant lessons at the start of every session
## 4. Verification Before Done
- Never mark a task complete without proving it works
- Run tests, check logs, demonstrate correctness in browser/simulator
- Ask yourself: "Would a staff engineer approve this?"
## 5. Demand Elegance (Balanced)
- For non-trivial changes: pause and ask "is there a more elegant way?"
- Skip for simple obvious fixes — never over-engineer
## 6. Autonomous Bug Fixing
- When given a bug: just fix it. No hand-holding
- Fix failing CI without being told how
I Why This Template Works
I Enforces planning before execution — dramatically reduces wasted effort
I Self-improvement loop is the core compounding mechanism
I Verification gate prevents "done" theater (claiming completion without proof)
I Elegance check prevents technical debt accumulation
I Autonomous bug-fixing eliminates dependency on human hand-holding

SECTION 05
Full Agentic Loop Protocol
Claude Code is designed as an agentic system. Making the loop explicit forces it to operate at maximum intelligence
and autonomy. This is the core of modern Claude Code operations as of February 2026.
Standard Agentic Operating Loop
GATHER
PLAN
ACT
VERIFY
REFLECT
IMPROVE
I  Loop until success criteria met
01
GATHER
Read CLAUDE.md, tasks/lessons.md, relevant files, errors, logs, and all context before touching any code.
02
PLAN
For any task > 30 min or with risk: enter Plan Mode, write a verifiable plan to tasks/todo.md.
03
ACT
Use tools and subagents. Make minimal, elegant, high-impact changes. One focused change at a time.
04
VERIFY RIGOROUSLY
Run tests / build / server / browser check. Use diffs when relevant. Never claim "done" without proof.
05
REFLECT
What worked? What failed? Why? Be honest about the root cause before moving on.
06
IMPROVE & LOOP
Immediately update CLAUDE.md and lessons.md with new rules. Repeat until success criteria are 100% met.
I Persistent Autonomous Mode (High-Performance Option)
I For bugs or long-running tasks: stay in the loop until the goal is fully achieved.
I "Try → fail → learn → try again." Do not exit early.
I Only stop after 3–5 solid attempts or when genuinely blocked by missing requirements.
I Break the loop and ask the user ONLY after multiple verified failures or for high-stakes architecture decisions.
